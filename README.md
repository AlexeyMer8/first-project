# Шпаргалка markdown

## Заголовки можно создавать с помощью символа `#`. Чем больше `#`, тем меньше заголовок.

## С помощью `---` можно добавлять черту под заголовками или над абзацем. Например:

Текст над чертой

---

Текст под чертой 

## Для разрыва строк используйте два пробела или сочетание символов `<br>`. Например:

Текст до переноса⋅⋅  
Текст после переноса <br>
Текст после второго переноса

Пример _курсива_ и **жирного** текста.

## Списки

Для оформления нумерованного списка достаточно поставить  цифры с точкой в начале. Например:

1. Первый пункт

2. Второй пункт

Для ненумерованного списка используют звездочку с пробелом в начале строки либо дефис с пробелом. Например:

* Первый пункт ненумированного списка

- Второй пункт ненумированного списка

## Ссылки

Чтобы сделать ссылкой часть текста, его заключают в квадратные скобки, а затем указывают нужный адрес в круглых скобках.

[Яндекс](https://www.yandex.ru)

Также можно добавить ссылке тайтл (от англ title — «название», «заголовок»). Тайтл — это всплывающая подсказка, которая появляется при наведении мыши на ссылку. Тайтл нужно заключить в кавычки и указать внутри скобок после адреса.

# Руководство работы с Git

## Команды для изменения конфигураций в Git

git config --global user.name "_имя юзера_" добавляет имя в конфигурацию гита файл git config

git config --global user.email "_имейл юзера_" добавляет имейл в конфигурацию гита файл git config

git config --list выводит содержимое конфигурации Git

## Команды перемещения и определения расположения файлов

cd / перейти в корневую директорию

cd ~ перейти в домашнюю директорию

cd .. перейти в родительскую директорию

cd _имя папки/имя папки2/_ перейти в указанную директорию

pwd выводит текущую директорию

ls вывести содержимое директории

ls -a просматривать содержимое директории вместе со скрытыми файлами и папками

&& с этим символом можно выполнить несколько команду одну за другой

↑↓ перемещении по истории команд в консоле

Tab дописывает выбранные пути, команды и др.

Tab + Tab если по введенному названию нет одного уникального подходящего варианта, то данная команда выведет все подходящие варианты

## Работа с файлами и папками

mkdir _имя папки_ создать директорию

touch _имя файла.формат_ создать файл

cp _имя файла.формат_ копировать файл

mv _имя файла.формат_ переместить (вырезать) файл

cat _имя файла.формат_ прочитать файл в консоле

## Удаление файлов и папок

rm _имя файла.формат_ удалить файл

rm -rf .git удалить подпапку .git (папка перестает быть репозиторием). -r рекурсивно (удаление папки и всего содержимого). -f принудительно без дополнительного подтверждения

rmdir _имя папки_ удалить директорию

rm -r _имя папки_ удалить директорию и все ее содержимое (если в директории есть папка с файлами, то она так же безвозвратно удалится с ПК)

## Работа с репозиторием

git init выолнив данную команду, папка в которой вы находитесь инициализируется и становится репозиторием

git status проверка статуса (состояния) репозитория, игнорируемые файлы не выводятся при команде git status

git status --ignored отображает все игнорируемые файлы

.git удалив эту папку на ПК можно разгитить нужную папку

git add _имя файла.формат_ подготовливает  указанный файл к сохранению

git add --all подготовит к сохранению все файлы

git add . добавит в репозиторий текущую папку со всеми файлами

echo "Любой текст" выведет в косоль переданный текст

echo "Любой текст" >> _имя файла.формат_ добавит указанный текст в конец в указанном файле

echo "Любой текст" > _имя файла.формат_ заменит весь текст в файле на записанный в ковычках

git diff показывает отличие измененного файла (modified) с его закоммиченным вариантом a/ текущая версия файла b/ измененная версия файла (modified). Цифры @@-1,2 +1,2@@  показывается что изменилось начиная с 1 строки 2 строки, минус и плюс нужны для чтения в черно-белых консолях

git diff --staged показывает отличие файла под наблюдением (staged после команды add) от закоммиченного варианта

git commit -m '_Ваш комментарий к коммиту_' делается коммит с указанным комментарием

git commit --amend --no-edit добавит к последнему коммиту все файлы под наблюдением, которые забыл добавить ранее, сообщение коммита не изменится, хэш изменится

git commit --amend -m "Новое сообщение" делает все то же, что и команда выше, но обновляет комментарий коммита

git commit --amend если случайно набрать данную команду, то запустится редактор

Esc + :qa! + Enter выход из редактора Vim (мой редактор)

Изменить текст сообзения + ctrl + X + Y (отобразится как ^X знак ^ обозначает Ctrl, а X означает exit) сохранит изменения, если N, то редактор nano закроется без изменения сообщения коммита

git restore --staged `имя файла.формат` отменяет добавление файла к наблюдению (откатывает add к modified или unetracked в зависимости от предыдущего состояния указанного файла)

git restore --staged `имя папки` отменяет все изменения файлов в папке добавленные к наблюдение (то же что и команда выше, только для всех файлов в папке)

git reset --hard `хэш коммита до которого нужен откат` безвозвратное удаление всех коммитов, которые были сделаны после указанного в команде коммита (указанный коммит становится последним)

git restore `имя файла.формат` отменяет изменения файла (возврат файла от modified к последнему состоянию в коммите или к последнему состоянию под наблюдением)

git push добавить изменения в удаленный репозиторий

git log показывает предыдущие коммиты сверху самые новые снизу самые старые

git log --oneline показывает сокращенный лог, удобно когда очень много коммитов было сделано ранее

q выход из просмотра логов, если он по каким то причинам не произошел автоматически

## Правила игнорирования файлов .gitignore

`#` с помощью решетки в начале текста можно оставить комментайри в gitignore и он не будет учитываться при игнорировании

.test_ignore игнорирование всех файлов с именем .test_ignore в корне и во всех папках репозитория

*.jpeg игнорировать все файлы, которые заканчиваются на .jpeg

docs/*/tmp игнорировать все файлы tmp во всех подпапках папки docs 

* символ звездочки будет игнорировать все файлы, так как под звездочку подойдет любое имя файла

file?.txt игнорирует файл с любым одним символом после текста file, например fileA.txt или file5.txt

file[0-2].txt квадратные скобки соответствуют игнорированию одного символа, из указанных в скобках, если символы написаны через знак `-`, то игнорируется символ из диапазона, можно применять как к цифрам так и к буквам [a-z]

file[abc].txt квадратные скобки соответствуют игнорированию одного символа, из указанных в скобках, если символы написаны подряд, то игнорируется символ при совпадении с указанным, можно применять как к цифрам так и к буквам [345]

/todo.txt слеш указывает на каталоги, если в gitignore в перед именем файла стоит /, то Git проигнорирует все файлы и каталоги только в корневой директории

build/ если слеш стоит в конце, то правило игнорирования применяется только к папке (файл с таким же наименование игнорироваться не будет)

docs/**/tmp парные звездочки работают как одинарные, отличие в работе с вложенными папками, двойная звездочка может соответствовать любому количеству таких папок (в том числе нулю), а одинарная соответствует только одной папке. Например данная команда проигнорирует и docs/old/tmp и docs/tmp и docs/old/saved/a/b/tmp

!doge.jpeg восклицательный знак делает исключение из правила, например у нас в gitignore была записана строка *.jpeg (данная команда игнорирует все файлы формата jpeg), но ! отменяется правило для конкретно наименования файла с форматом jpeg, так что файл doge.jpeg будет отслеживаться. Если была прописана строка *.log, то можно так же применить !examples/**/*.log  это отменяет игнорирование файлов .log в папке examples

## Как читать git status

Существует 4 основных статуса в git:

staged все изменения файла добавлены к наблюдению (после git add файл не изменялся)

modified файл изменен, но не добавлен к наблюдению (нужно прописать git add, чтобы изменения учитывались при коммите)

untracked файл новый и никогда не добавлялся к наблюдения (нужно прописать git add, чтобы файл добавился к наблюдению)

tracked статус который по умолчанию присваивается всем, кроме untracked, но данный статус указывается неявно (не выводится при git status)

## Правила оформления коммита:

- Короткое, но информативное сообщение на русском

- Первое слово глагол в прошедшем времени `Исправил, добавил и т.д.`

- Для коммитов на английском используют повелительное наклонение (imperative): `Use library mega_lib_300`

- Если есть Jira-ID, то обязательно указать: `LGS-259: Добавил инфо о правилах коммита`

- Для репозитория с исходным кодом использовать feat для новой функциональности и fix для исправления ошибок: `feat: текст коммита`

- В GitHub можно использовать его стиль с помощью `#`: `Исправить #334, добавить график температуры`
